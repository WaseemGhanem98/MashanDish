# Catering App

Mobile-first PWA where org members log in, see today's vendor menu, and pick one
meal before a daily cutoff. At cutoff the order is locked and emailed to the
vendor. Built with Next.js 16 (App Router) + Supabase + Resend, deployed to
Vercel.

See [`docs/PRD.md`](./docs/PRD.md) for the full product spec.

## Stack

- **Next.js 16** App Router, TypeScript strict, React Server Components
- **Tailwind v4** + **shadcn/ui** (New York / neutral)
- **Supabase** for auth + Postgres + RLS, accessed via `@supabase/ssr`
- **Resend** for vendor emails (Slice 7+)
- **Vercel** hosting + Cron (Slice 7+)

## Project layout

```
app/
  (auth)/login/         # public login page
  (member)/             # auth-gated member pages
  admin/                # admin role-gated pages
  auth/sign-out/        # POST route handler
  forbidden.tsx         # 403 page (Next 16 authInterrupts)
  manifest.ts           # PWA manifest
components/
  ui/                   # shadcn primitives
  auth/                 # sign-out button, etc.
lib/
  supabase/             # server / client / admin / proxy factories
  auth.ts               # cached getCurrentUser / getCurrentProfile
  utils.ts              # cn()
proxy.ts                # session refresh on every request (formerly middleware.ts)
supabase/
  migrations/           # forward-only SQL migrations
  seed.sql              # bootstrap org + first admin
types/database.ts       # generated by `supabase gen types`
```

## Setup

### 1. Install dependencies

```bash
npm install
```

### 2. Create a Supabase project

1. Sign up at [supabase.com](https://supabase.com) and create a new project.
2. From **Settings → API**, copy the values into a new `.env.local` (already
   created from `.env.example` if you cloned this repo cleanly):

   ```bash
   cp .env.example .env.local
   ```

   Fill in:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` (accepts `sb_publishable_...` or legacy anon key)
   - `SUPABASE_SECRET_KEY` (accepts `sb_secret_...` or legacy service_role key)

### 3. Push the schema

```bash
npx supabase login
npx supabase link --project-ref <your-project-ref>
npx supabase db push
```

### 4. Bootstrap your first admin

1. Sign up your admin email through the Supabase dashboard
   (**Authentication → Users → Add user**) or via the `/login` page once the
   dev server is running.
2. Edit `supabase/seed.sql` — change `admin@example.com` to your email and
   adjust the org name / vendor email / timezone.
3. Run the seed in the Supabase SQL editor (or via `npx supabase db reset` if
   you're on local).

### 5. Regenerate database types

After every schema change:

```bash
npx supabase gen types typescript --linked > types/database.ts
```

### 6. Run the app

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000). Signed-out hits go to
`/login`; signed-in members land on `/`; admins can also reach `/admin`.

## Build slices

The app is built in vertical slices — see `docs/PRD.md` § Build Order. Slice 1
(this README's scope) is auth + profiles + protected routes. Subsequent slices
add menu CRUD, selections, the cutoff job, vendor email, exports, and PWA
polish.

## Conventions

Project-wide rules live in `.cursor/rules/`. The big ones:

- Server Components by default; client components only for interactivity
- Every table has RLS enabled; never disable it "just to test"
- Auth via `@supabase/ssr` (cookies are async in Next 16)
- Session refresh in `proxy.ts` (renamed from `middleware.ts` in Next 16)
- No hardcoded colors — use the shadcn theme tokens in `app/globals.css`
